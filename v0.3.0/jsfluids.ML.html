ML

Class for dealing with data from ML models. You need a VTK Unstructured Grid (.vtu) and an ONNX model. ONNX models can be converted from several ML frameworks. See https://github.com/simzero/cfdonnx for more details.

var model = jsfluids.ML;

Methods

SDFAndRegion(buffer) → {Float64Array}

Description:

  • Computes the signed distance field (SDF) and flow region fields on the grid based on an STL.

    • It assumes the grid has a SDF2 field with the SDF from the top/bottom surfaces and flowRegion with inlet, outlet and walls.
    • It creates a new SDF1 field with the SDF from the walls defined from the STL.
    • It modifies the flowRegion field to label the field inside the STL with 0.

    Note: This function is optional and specifically designed for DeepCFD.

Example
var sdf = model.SDFAndRegion(stlBuffer);
Parameters:
Name Type Description
buffer Buffer

The STL buffer
Returns:

The array with fields SDF1, flowRegion and SDF2.

Type
Float64Array

(async) loadMesh(mesh)

Description:

  • Loads a VTK Unstructured Grid (.vtu) mesh from either an URL or a buffer.

    This function initialize the module and the scene, and sets the number of cells.

Example
model.loadMesh(meshURL).then(() => {
  ...
});
Parameters:
Name Type Description
mesh string | Buffer

The mesh to load, which can be either an URL or a buffer of a VTU file:

  • If mesh is a string, it is treated as an URL and the mesh will be loaded from the URL.
  • If mesh is a buffer, the mesh will be loaded from the buffer.

setComponent(dict) → {string}

Description:

  • Sets a visualization component and returns a string representation of a GLTF

Properties:
Name Type Attributes Description
dict.component string

The component to visualize. Must be one of "surface", "plane", or "streamlines".
dict.planeProperties.origin Array.<number> <optional>

Required when component is "plane. The origin vector, which is an array with three numbers representing the x,y,z coordinates.
dict.planeProperties.normal Array.<number> <optional>

Required when component is "plane. The normal vector, which is an array with three numbers representing the x,y,z coordinates.
dict.streamlinesProperties.field string <optional>

Required when component is "streamlines". The field to calculate the streamlines.
dict.streamlinesProperties.center Array.<number> <optional>

Required when component is "streamlines. The center of the streamlines seeding sphere, which is an array with three numbers representing the x,y,z coordinates.
dict.streamlinesProperties.radius number <optional>

Required when component is "streamlines". The radius of the streamlines seeding sphere.
dict.streamlinesProperties.propagation number <optional>

Required when component is "streamlines". The propagation lenght of the streamlines.
dict.streamlinesProperties.tubeRadius number <optional>

Required when component is "streamlines". The radius of the streamlines tube.
dict.streamlinesProperties.tubeSides number <optional>

Required when component is "streamlines".The number of sides for streamlines tubes.
dict.streamlinesProperties.resolution number <optional>

Required when component is "streamlines". The numbers of streamlines to generate.
Example
var gltfSurfce = model.setComponent({
  type: "surface"
});

var gltfPlane = model.setComponent({
  type: "plane",
  planeProperties: {
    origin: [0, 0, 0],
    normal: [1, 0, 0]
  }
});

var gltfStreamlines = model.setComponent({
  type: "streamlines",
  streamlinesProperties: {
    field: "U",
    center: [0, 0, 0],
    radius: 1.0,
    resolution: 8,
    tubeRadius: 0.1,
    tubeSides: 30,
    propagation: 300.0
  }
});
Parameters:
Name Type Description
dict Object
Returns:

The GLTF data as a string
